REGLAS DE PROCESO PARA NUEVAS TAREAS / MÓDULOS: Apliocable solo para una tarea / proceso / módulo / o sección nueva
- Analiza por completo la solicitud
- Muestra aspectos importantes a considerar antes de proceder con el plan
- Comenta de la información necesaria para ejecutar ese proyecto
- Valida los requerimientos y ayuda a que sean robustos y técnicos
- Sugiere un idioma recomendado para ejecutar un plan posterior a esta primer fase de discovery [Guiarse con el rule: ai-model-selection.md]

REGLAS DE PROCESO PARA CORRECCIÓN DE BUGS O ERRORES:
- Siempre provee un RCA completo y detallado
- Valida los requerimientos técnicos para saber si algo está distinto a lo esperado
- Sugiere un idioma recomendado para ejecutar un plan posterior a esta primer fase de discovery [Guiarse con el rule: ai-model-selection.md]

REGLAS GENERALES:
- Siempre espera los inputs, recomendaciones y aprobación del usuario para proceder a ejecutar cambios.
- Solo tienes permiso a ejecutar de una vez si te digo [ EJECUTA DE UNA VEZ ] o algo similar.

EXCEPCIONES:
- Los Workflows ejecútalos de inmediato, no generes un plan ni solicites permiso
- Los Workflows no necesitan que especifiques que modelo de IA utulizar [Ignora el rule: ai-model-selection.md]
